Phase 2 — Prioritas Utama Integration Testing — Auth & Security ~8 tests · mulai di sini ▼
Setup sekali: Testcontainers (PostgreSQL + Redis) + MockMvc. Semua test berikutnya pakai fondasi yang sama.
Setup — lakukan sekali
BaseIntegrationTest.java
Abstract class dengan @SpringBootTest, Testcontainers PostgreSQL + Redis, @Transactional untuk isolasi data antar test
Auth flow — paling kritis
POST /auth/register → 201
User baru terdaftar, token dikembalikan, refresh token tersimpan di DB
POST /auth/login → 200
Login berhasil, access token valid, HttpOnly cookie set
POST /auth/register duplikat → 409
Email yang sudah ada melempar ConflictException dengan format ErrorResponse yang benar
Security config — hal yang unit test tidak bisa verifikasi
GET /products — tanpa token → 200
Public endpoint bisa diakses tanpa JWT. Verifikasi SecurityConfig benar.
GET /cart — tanpa token → 401
Protected endpoint ditolak tanpa JWT. AppAuthenticationEntryPoint bekerja.
GET /cart — dengan token valid → 200
JWT filter bekerja end-to-end: parse token → load user → authorize
POST /admin/... — sebagai CUSTOMER → 403
Role-based access control bekerja. AccessDeniedHandler mengembalikan format yang benar.
POST /auth/login 11x → 429
Rate limiter benar-benar bekerja di skenario nyata dengan Redis. Verifikasi header Retry-After.
Phase 3 Integration Testing — Core Business Flow ~6 tests ▼
Test end-to-end flow yang mencerminkan use case nyata user. Setiap test mensimulasikan journey satu user dari awal sampai akhir.
Happy path flows
Full order flow
Register → Login → POST /cart → POST /orders → GET /orders/{id}. Verifikasi stock berkurang, snapshot tersimpan, response correct.
Cancel order → stock restored
Buat order → cancel → verifikasi stock product kembali ke jumlah semula di DB nyata
Product search dengan filter
GET /products/search?categoryId=1&minPrice=0&maxPrice=20000000 — verifikasi query JPQL benar
Error path yang penting
Checkout dengan stock habis → 409
Product stock = 0 saat checkout, ConflictException dengan format ErrorResponse benar
Order milik user lain → 404
GET /orders/{id} dengan id order milik user lain. Ownership check bekerja di DB nyata.
Request body tidak valid → 400
POST /orders dengan cartItemIds null → 400 dengan validation error message dari @Valid
